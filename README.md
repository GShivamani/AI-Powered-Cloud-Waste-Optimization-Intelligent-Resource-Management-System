## Overview

Cloud environments often suffer from significant resource waste — idle EC2 instances, underutilized S3 buckets, and inactive RDS databases silently accumulate costs. This project provides an **automated, AI-powered pipeline** to:

- **Simulate** cloud resource usage data
- **Detect** wasteful resources based on usage thresholds
- **Classify** resources using K-Means clustering (AI)
- **Recommend** automated actions (DELETE / RESIZE)
- **Learn** from user feedback to improve accuracy over time
- **Track** savings trends with historical reporting

## Modules

### 🔹 Module 1 — Cloud Resource Collector (`cloud_waste_collector.py`)
Simulates a pool of **30 cloud resources** (EC2 Instances, S3 Buckets, RDS Databases) and generates realistic usage metrics.

| Resource Type | Metric | Waste Threshold |
|---|---|---|
| EC2 Instance | CPU Utilization | < 50% |
| S3 Bucket | Storage Utilization | < 60% |
| RDS Database | Active Connections | < 10 |

- Assigns random states: `idle`, `underused`, `critical`
- Flags each resource as `waste: YES/NO`
- Saves output to `cloud_usage_report.csv`

---

### 🔹 Module 2 — Analyzer & Visualizer (`cloud_waste_analyzer.py`)
Reads the CSV generated by Module 1 and produces:

- **Waste Summary Table** — counts and percentage by resource type
- **Bar Chart** — Waste % per resource type
- **Histogram** — Usage value distribution across all resources

---

### 🔹 Module 3 — AI Classifier (`ai_classifier.py`)
Uses **K-Means Clustering** to automatically classify resources based on CPU and Memory utilization.

| Cluster Tag | Description |
|---|---|
| `IDLE_DELETE_CANDIDATE` | Very low usage — safe to delete |
| `UNDERUSED_RESIZE_CANDIDATE` | Moderate usage — recommend downsizing |
| `CRITICAL_HIGH_USAGE` | High usage — no action needed |

- **Potential monthly savings** are estimated per cluster:
  - Idle: 100% cost savings
  - Underused: ~50% cost savings (downsize)

---

### 🔹 Module 4 — Insights Dashboard & Automated Actions (`waste_manager_insights.py`)
Generates an executive-level insights report and a prioritized action list.

**Report includes:**
- Total resources monitored
- Classification breakdown
- Financial impact (estimated monthly savings)

**Automated Actions generated:**
- `DELETE` — for idle resources with CPU < 1.5%
- `RESIZE` — for underused resources with CPU < 15%
- All actions flagged as `PENDING_APPROVAL` before execution

---

### 🔹 Module 5 — Continuous Learning & Feedback Loop (`feedback_loop.py`)
Simulates **user approval/rejection** of recommended actions to enable AI retraining.

**Workflow:**
1. Top 10 high-impact actions are presented for review
2. User rejects misclassified actions (e.g., a resource flagged as IDLE that is actually critical)
3. Corrected labels are saved as **new ground truth** training data
4. Next iteration retrains K-Means with corrected data → improved accuracy

---

### 🔹 Module 6 — Trend Analysis & Reporting (`trend_reporter.py`)
Generates **3-month historical reports** to visualize the system's effectiveness over time.

- **Line Chart 1** — Realized monthly cost savings trend (↑ increasing over time)
- **Line Chart 2** — Total waste resource count trend (↓ decreasing over time)
- Simulates 20% monthly reduction in waste cost and 15% reduction in waste resource count

## How It Works

**The big idea:**

> *"This system collects cloud resource data, detects waste, classifies resources with AI, recommends cost-saving actions, incorporates user feedback, and tracks trends to optimize cloud usage efficiently."*

The pipeline is designed to be a **closed-loop, self-improving system**:

1. Data is collected and wasteful resources are flagged automatically
2. AI classifies every resource without manual intervention
3. Cost-saving actions are generated and ranked by financial impact
4. Human feedback corrects misclassifications and feeds back into the model
5. Over months, waste decreases and savings compound — as visualized in trend reports
